C/C++ 学习笔记(Windows) 内存与指针

##内存##

  • 内存是由按顺序编号的一系列存储单元组成的，每个存储单元（8位，一个字节）有唯一的地址，通过地址可以方便的在内存单元中存储信息。
  • 内存与CPU读写速度快，断电就没有了，容量比较小，成本比较高
  • 高级服务器一般将读写很频繁的数据存储在内存中
  • 一个程序载入内存，代码数据都有地址；外挂就是调用函数，修改数据，函数就是代码，数据就是变量。

##指针##

  • 变量是对程序中数据存储空间的抽象
  • 指针：一个变量的地址（首地址，如果解析变量的值，由指针的类型决定）；指针变量：专门存放变量地址的变量
  • int num=10; &num 是num的地址；*(&num)是num的值
  • 指针变量的定义：数据类型 *指针变量名=指向的变量的地址；
  • 栗子：int *p=# 这里指针变量p指向一个int型变量num。指针变量p中存储着num的地址，注意只是起始地址，那么用指针读取变量内容时，读取几个字节呢？就是由指针变量的类型决定的。比如这里是int型的，那么就读取4个字节的内容。
  • p就是p指向的变量num的值，对p赋值就相当于对num赋值
  • 指针使用之前必须初始化
  • 空指针是不指向任何变量的指针，int *p=NULL，但空指针指向的内容为系统所用，不能随便访问; NULL在stdio.h中定义为常量。
  • 直接访问与间接访问。直接访问：按变量地址存取变量值；间接访问：通过存放变量地址的变量去访问变量。

###指向指针的指针###
指针变量也是变量，占据一定的内存空间，有地址，因此可以用一个指针指向它，称为指向指针的指针或二级指针。
例如：double num; double p=# double *pp=&p;

###指针的类型###

  • 类型代表了所占内存的大小和对二进制数的解析方式
  • 不同类型的指针不能相互赋值，否则会出现解析错误
  • 不管什么类型的指针，在32位系统下都占4个字节，与类型无关，但怎样解析指针指向的地址的内容就与指针类型有关了

###指针变量的值###
是指指针本身存储的数值，这个值将被编译器当做一个地址（所指向的数据所在内存的首地址），而不是一般的数值。

###野指针与空指针###

  • 没有初始化的指针是野指针。如果在初始化之前使用该指针，一般会导致程序崩溃
  • 初始化为NULL的指针为空指针，指向0x00000000。没有权限访问空指针所指向的内容，因为该地址是操作系统使用的。

###指针的运算###

指针变量的赋值运算

  • p=&a; 将变量a的地址赋给p
  • p=array; 数组arra的首地址赋给p
  • p=&array[i]; 数组元素的地址赋给p
  • p1=p2; 指针变量p2的值赋给p
  • 不能把一个整数赋给指针变量，也不能把指针变量的值赋给一个整型变量

指针变量的算术运算

  • 指针的自增(++)和自减(–)，每次前进或后退sizeof(指针类型)个字节数
  • 指针的加减法在非数组内部没有意义，容易越界报错（因为一个进程不能读写其他进程的内存）
  • 若p=a：
  • p++与(p++)等价，先引用(取出*p的值)再自增(P=p+1)
  • ++p先自增再引用，(*p)++表示p所指向的元素值加1，指针值，即所指向的地址不变
  • (p++)与(++p)，前者是先去p的值，后使p值加1，相当于a[i++]；后者先使p加1，再取p的值，相当于a[++i]

###指针的比较###
p1==p2 为真，说明两个指针指向同一地址；否则，两个指针不指向同一地址
p1>p2 p1<p2 比较两个指针所指向的地址的大小。在数组内部是比较地址靠前还是靠后

###指针相减###

  • 两个同类型的指针相减，返回值是一个有符号整数。它们之间并不是简单作差，还要除以指针所指类型占用的内存字节数。所以这个有符号整数指示的是两个指针所指向的元素之间间隔的元素数。
  • 指针相减一般用于同一片内存空间，如数组。不同内存空间的指针相减没有什么实际意义。

###函数指针###
如果在程序中定义了一个函数，在编译时，编译系统为函数代码非配一段存储空间，这段空间的起始地址，称为这个函数的指针，可以定以一个指向函数的指针变量指向该函数的起始地址。
定义的一般形式：数据类型 (*指针变量名)(参数参数表列)=指向的函数名;
指向函数的指针变量一个重要用途：是把函数的地址作为实际参数传递到其他函数的形式参数（指向函数的指针）。

###指针变量有关的一些扩展概念###
指针变量与左值
指针变量以及指针变量的间接引用都可以作为左值。
例如：int num1=0; int p=&num1; p=1;

指针与整型

  • 无论指针指向什么类型的变量，对32位系统来说，都占4个内存字节，指针的值是某个内存的地址，是个整数。
  • 如果要对某个内存地址进行访问，可以通过强制类型转换来完成。如：int p=(int )0x0012FF7C;

void指针与空指针
void 指针是一种特殊的指针，不指向任何类型的数据。
用于参数还有返回值，不明确指针类型的情况下传递地址需要用到空类型的指针。
如果需要用此地址指向某类型的数据（可以接收任意类型数据的地址），应先对地址进行类型转换。
例如：int num=10; void p=# printf(“%d”,((int )p));//这样可以取出相应地址的值（printf(“%d”,p);这样是不行的，因为void类型不明确，不知道该取出几个字节）

空指针是不指向任何地址的指针，值为NULL。

###内存操作###
memset()

动态分配内存和静态分配内存

  • 静态内存非配是在编译阶段（编译阶段将代码翻译成机器码）完成的，如声明数组时要明确指出数组大小。但缺点是分配过大或内存不足，因此引入动态分配机制。注意：数组只能处理小数据，大了会溢出
  • 动态内存分配是在运行阶段根据需要分配或释放内存，大小可以控制。使用完毕后，可以申请释放，由系统回收。

malloc()
malloc和free是C标准库stdlib.h提供的两个函数，用以动态申请和释放内存。
调用格式：
void *malloc(unsigned int size);

  • 参数size是无符号整数，表示申请内存的大小。执行成功，系统会为程序开辟一块大小为size个字节的区域，并将区域的首地址返回，用户可利用该地址管理并使用该块内存；申请失败，返回空指针NULL。
  • 返回类型是void *，用以返回值对其他类型指针赋值时，必须进行显示转换。
  • 申请内存的长度可以通过“长度*sizeof（类型）”来计算。

free是释放内存，如free(p);
注意：free根据地址释放内存，用free释放过的内存不能重复释放，否则出错。空指针可以重复释放多次，不会出错。

calloc
void *calloc(size_t num, size_t size);
calloc与malloc的区别：

  1. 参数不同，calloc有两个参数，第一个参数是对象个数，第二个参数是对象占据的内存字节数。
  2. malloc以字节为单位申请内存；calloc以目标对象为单位分配，目标对象可以是数组，可以是结构体。
  3. calloc分配完内存之后，将内存的内容初始化为0；malloc分配内存，不进行初始化。

realloc
重新分配内存，如果可以在原来分配的内存空间下面拓展空间，则直接拓展，在原来地址的后面增加内存；否则先回收以分配的内存，然后重新分配内存空间
(void )realooc((void p),sizeof(int)*(num+addnum))
第一个参数：原来分配的内存的地址
第二个参数：现在需要的内存数

动态分配内存注意事项

  • 指向动态分配内存的指针在释放之后的指针的值不变，但存储的数据都已清空，如果再次引用指针访问内存内容，得到垃圾数据
  • 在释放内存以后，按照软件工程的规范，指针应赋值为空，这时再引用指针会直接报错，可以快速定位错误。
  • 如果指向内存的指针消亡或者指针的值发生变化，对应的内存不会被释放，会引起内存泄露

##x86和x64##
32位计算机

  • 最大只能识别4G内存
  • 因为地址为32位，最大为0x1111 1111 1111 1111 1111 1111 1111 1111=（4294967295）10,4294967295/1024/1024/1024=4G

64位计算机

  • 地址为64位
  • 服务器级编程需要64位服务器，有海量内存
  • 64位系统可以运行32,64位应用程序；32位只能运行32位程序；ARM只能进行远程调试。

##VS2013中debug和release##

  • debug附加了很多调试信息，主要用于调试，生成的文件较大
  • release经过了代码优化，去掉了调试信息，主要用于发布，生成的文件较小，且速度要快于debug

##指针复习##
一级指针：

  • 函数内部间接修改外部变量；
  • 跨进程修改变量（外挂）；
  • 数组作为参数，可以用一级指针作为形式参数接收数组的首地址。
  • 可以存储一个数组的首地址，可以用指针的方式和下标的方式访问数组
  • 可以作为函数返回值返回地址，一定不能返回指向栈的地址
  • 间接访问结构体，共用体，枚举体，创建一个堆上的变量，存储这个变量的地址

函数的返回值有副本机制，函数执行完后，返回值地址被回收，值存储在寄存器后者能够，所以函数返回值不能够取地址

注意：定义指针变量必须初始化，否则，会出错（使用未初始化的变量，但这是在VS2013中，它内部有严格的安全检查机制，如果在命令行中是可以编译通过的）。可以以这种形式初始化：int m; int *p=&m;
